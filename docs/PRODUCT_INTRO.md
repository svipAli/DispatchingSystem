# 🚀 一站式 MCP 服务调度与计费平台

## 一句话概括

**把几十台 MCP 服务器变成统一 API 服务，让 Dify 智能体一键调用，自动计费扣费。**

---

## 解决什么问题

### 痛点 1：MCP 服务散落各处，无法统一管理
你有几十台机器跑着各种 MCP 服务——文本生成、代码执行、图片处理、WordPress 操作……每台机器独立运行，没有统一入口。

**→ 我们的平台：** 所有节点注册到平台，自动发现服务，统一对外提供 API。

### 痛点 2：智能体平台不知道怎么调用
Dify、Coze 等 AI 智能体需要标准 MCP 协议接入。手动对接每个服务太麻烦。

**→ 我们的平台：** 一键对接 Dify，填个 URL 就能把所有服务暴露给智能体，自动识别参数。

### 痛点 3：计费混乱，无法收款
用户调了多少次？每次多少钱？谁欠费了？靠人工记账根本不可能。

**→ 我们的平台：** 自动预扣费、失败退款，完整的充值/消费/流水记录，清清楚楚。

### 痛点 4：运维全靠人工
哪个节点挂了？负载多高？CPU/内存够不够？

**→ 我们的平台：** 可视化监控面板，节点状态实时掌握，AI 助手辅助排查。

---

## 核心亮点

| 亮点 | 说明 |
|------|------|
| 🔌 **即插即用** | 节点装上就能用，自动注册服务，零配置 |
| 🤖 **AI 原生** | 内置 AI 运维助手，自然语言对话管理平台 |
| 💰 **精准计费** | 按次定价、预扣费、失败退款，流水可追溯 |
| 📊 **实时监控** | ECharts 仪表盘，CPU/内存/磁盘一目了然 |
| 🔐 **安全可靠** | JWT 鉴权、Token 黑名单、管理员 RBAC 权限 |
| 🚀 **部署简单** | 一条命令部署，Docker 或裸机都行 |

---

## 适合谁用

- **MCP 服务提供商** — 把你的 AI 服务打包成商品，按次收费
- **企业内部平台** — 统一管理所有内部 MCP 节点，分部门计费
- **Dify 用户** — 一键对接，让 Agent 调用任何自定义服务

**把你的 MCP 服务变成可售卖、可管理、可监控的 SaaS 产品。**
